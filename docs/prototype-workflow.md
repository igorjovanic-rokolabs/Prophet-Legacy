# Prototype Workflow

Use this document as the working context while recreating the platform from screenshots.

## How To Send Screenshots

For each screen, send:

- The screenshot.
- The screen name if you know it.
- What the user is trying to accomplish.
- Any important role, permission, state, or data condition.
- Which screen comes before and after it in the workflow, if known.

## What Will Be Captured

Each recreated screen should include:

- A React screen component.
- Notes about primary user goal and key interactions.
- Any visible states such as loading, empty, error, disabled, selected, expanded, or modal open.
- Connections to previous and next screens.
- Open questions for the team.

## Design Direction

The prototype should follow the original platform closely for UX and layout, while using shadcn/ui as the visual language. This means the same structure and behavior, but cleaner token-based styling that is easier to maintain and later recreate in Figma.
